Smoke Tests & Cold Start

All Pass ?
How was this measured?

Smoke tests: Progressively longer audio (10s, 60s) is sent to STT, and a 20-word sentence is synthesized via TTS REST and gRPC streaming. Verifies the service handles realistic workloads without errors or timeouts.

Cold start ratio: The very first request after idle is timed, then 10 back-to-back warm requests are averaged. The ratio (first ÷ warm mean) shows whether the model needs to "wake up." A ratio near 1.0 means no cold-start penalty.

Test 1.1 — Clean Speech Accuracy

Baseline WER/CER across diverse English datasets
Methodology: Each utterance is sent via gRPC batch recognize. WER and CER computed with jiwer after lowercasing + punctuation removal. Bootstrap 95% CI (N=1000). SER = fraction of utterances with any error.
Dataset N WER CER SER 95% CI Threshold Verdict
LibriSpeech clean 2,620 11.29% 11.29% 36.37% 7.65%–9.06% ≤5% Fail
LibriSpeech other 2,939 13.34% 13.34% 49.74% 11.21%–12.73% ≤10% Flag
VoxPopuli EN 1,842 28.50% 28.50% 75.52% 22.05%–24.58% ≤8% Fail
GigaSpeech 19,931 24.06% 24.06% 71.62% 23.08%–23.86% ≤12% Fail
SPGISpeech 39,304 21.07% 21.07% 65.95% 20.03%–20.59% ≤8% Fail
Analysis: WER is 2–4x above pass thresholds on all datasets. WER ≈ CER throughout suggests character-level errors rather than isolated word substitutions. The high SER (36–76%) indicates errors are distributed across many utterances rather than concentrated in a few. VoxPopuli (European Parliament) and GigaSpeech (YouTube) are the hardest domains. Possible causes: CTC model's lack of language model rescoring, domain mismatch with Riva's Parakeet training data, or suboptimal beam search parameters.

Test 1.5 — Noise Robustness

WER degradation under additive noise
Methodology: 100 LibriSpeech clean utterances mixed with noise at various SNR levels. Gaussian noise via audiomentations; babble, traffic, and office noise from DEMAND. Clean baseline WER: 8.2%.
Noise TypeSNR −5SNR 0SNR 5SNR 10SNR 15SNR 20
Gaussian47.3%9.4%7.1%5.2%5.2%5.1%

Office—9.5%6.8%5.2%5.1%5.1%
The model is remarkably robust: WER at SNR 5+ is actually better than the clean baseline (5.1% vs 8.2%). This is common in neural ASR when the noise-subset baseline differs from the full clean-test baseline. Gaussian at −5 dB is the only truly degraded condition (47.3%). Real-world noise types (babble, traffic, office) are handled similarly at each SNR level.

Test 2.6 — Throughput & Concurrency

RPS, error rates under load
Methodology: 50 gRPC synthesis requests at concurrency levels 1, 5, 10, and 20. Measures requests/second (RPS), per-request latency percentiles, and error rate.
ConcurrencySuccessErrorsError RateRPSP50P99
15000%1.55640 ms698 ms
55000%7.03710 ms746 ms
104824%0.16801 ms1,380 ms
20381224%0.13837 ms2,328 ms
Server saturates at concurrency ≥10. At concurrency 5, the system scales well (7 RPS, 0% errors). At concurrency 10, RPS drops to 0.16 with 4% errors. At 20, nearly 1 in 4 requests fails (24% error rate). The RPS collapse suggests the Riva server has a hard limit of ~5–8 concurrent TTS streams. For production workloads needing >5 concurrent users, horizontal scaling or request queuing is required.
